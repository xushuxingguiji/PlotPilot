叙事主线

1故事：要解决什么，如何闭环

矛盾：长篇创作周期长，设定、结构、伏笔、文风容易漂移。解法：把「结构树 + 章节正文 + 设定与知识」放进可查询的存储，并让每次规划/写作/保存都能读旧写新、留下痕迹。闭环：规划产出可执行的章节点 → 写作落库 → 自动沉淀索引与图谱 → 下一波生成与审稿吃同一份数据，回到规划或继续写，形成螺旋上升。

核心定位

结构化规划 + 人机协作写作 + 一致性工具（伏笔、元素、文风、审稿、上下文检索）。

两条入口，一套模型

Web：工作台经 FastAPI 写 SQLite/文件。CLI：离线流水线写 output/novels/<slug> JSON/Markdown。二者共享 ARK_*，叙事上视为「同一创作模型」的不同交付面。

闭环检验问题

任意时刻问：当前章节依赖的设定从哪读？保存后谁更新索引/图谱？下一章生成上下文从哪拼？下图用一条链回答。

副线闭环

5保存章节：主事务 + 副线扇出

实现见 interfaces/api/v1/chapters.py：正文写入成功后，索引/图谱/文风等尽力而为，失败记日志不拖垮主路径——这是工程闭环，不是业务省略。

flowchart LR
    PUT[PUT chapters 正文]
    TX[事务写 SQLite 章节]
    PUT --> TX
    TX --> OK[200 返回前端]
    TX --> BG[副线扇出]
    BG --> I[向量摘要索引]
    BG --> G[KG infer 若可解析 story_node]
    BG --> DR[文风 drift score]
    I --> CLOS[供 search 与 retrieve]
    G --> CLOS
    DR --> CLOS

逻辑闭环：副线产物必须能在「知识检索 / 上下文组装 / 审稿」中被消费；若某环节未部署（如无向量服务），则对应箭头静默跳过，主写作仍成立。

能力域拆解

7能力域：右侧栏在管什么

每块能力都应对「规划 / 写作 / 检查」某一时刻的痛点；与第 9 节矩阵互证。

1. 故事线 (Storylines)

🎯 设计目标

管理小说的主线、支线与暗线，规划故事线的起止章节和关键里程碑。

前端组件: StorylinePanel.vue

位置: 右侧面板 → 创作设定 → 故事线

功能:

  • 添加故事线（类型：主线/爱情线/复仇线/悬疑线等）
  • 设置章节范围（estimated_chapter_start ~ estimated_chapter_end）
  • 查看故事线状态（进行中/已完成/已废弃）
  • 编辑与删除故事线

2. 情节弧线 (Plot Arc)

🎯 设计目标

设计小说的情节张力曲线，规划起承转合的关键剧情点和节奏控制。

前端组件: PlotArcPanel.vue

位置: 右侧面板 → 创作设定 → 情节弧

可视化: SVG 绘制张力曲线，章节号为 X 轴，张力值为 Y 轴

3. 时间线 (Timeline)

前端组件: TimelinePanel.vue

位置: 右侧面板 → 创作设定 → 时间线

存储: Bible 的 timeline_notes 字段

4. 伏笔账本 (Foreshadow Ledger)

🎯 核心亮点：感官锚点匹配

输入当前场景的感官描述（视觉/听觉/嗅觉等），AI 自动找出最匹配的待兑现伏笔。

前端组件: ForeshadowLedgerPanel.vue

位置: 右侧面板 → 创作设定 → 伏笔

双 Tab 设计:

  • 待兑现: 显示所有 pending 状态的伏笔 + 感官匹配功能
  • 已消费: 显示所有 consumed 状态的伏笔

8持续规划：边写边长出的结构树

核心理念

不必一次性规划全书：先宏观定骨架，再在幕内展开章节；写作推进后由 continue 判断是否需要新幕，形成结构随稿生长的外环，与第 1 节主闭环嵌套。

完整工作流程

阶段 1: 初始规划

  1. 设计故事线（主线/支线）
  2. 设计情节弧（张力曲线）
  3. 设计时间线（关键事件）
  4. 宏观规划（部-卷-幕）
  5. 幕级规划（第一幕的章节）

阶段 2: 写作循环

  1. 写作前：查看伏笔、章节元素、对话沙盒
  2. 写作章节：编辑器中写作
  3. 写作后：添加伏笔、关联元素、计算文风
  4. AI 续规划：自动判断是否需要下一幕
  5. 创建下一幕：规划新幕的章节

阶段 3: 定期检查

  • 每 5-10 章：文风漂移、伏笔状态
  • 每完成一幕：情节弧、时间线
  • 每完成一卷：重构扫描

关键技术点

🎯 黄金比例分配

宏观规划时，按 1:1.618:1 分配章节数到各部，确保结构平衡。

🔒 智能合并

确认宏观规划时，保留已写章节，删除空节点，避免数据丢失。

🤖 AI 续规划

写完章节后自动判断，是否需要创建下一幕，实现渐进式规划。

🔍 感官锚点匹配

通过感官描述（视觉/听觉/嗅觉）匹配伏笔，提升伏笔兑现率。

📊 文风指纹

通过形容词密度、平均句长等指标，量化文风，监控漂移。

🔗 元素关联

章节与 Bible 元素的关联，支持正向查询和反向查询。
